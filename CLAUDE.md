# 项目协作规则

这是一个基于现有模板修改的个人博客项目。请在处理本项目时遵守以下规则：

## 总原则

1. 以最小改动为原则，只修复当前任务相关问题。
2. 不要顺手重构，不要大范围格式化整个仓库，不要修改无关文件。
3. 不要轻易改动项目现有结构、命名、组件逻辑和页面效果，除非任务明确要求。
4. 尊重项目现有代码风格和组织方式，优先延续已有实现。

## 报错修复规则

1. 如果任务是修复 CI、lint、typecheck 或 build 报错，优先只修改报错涉及的文件。
2. 修复前先阅读报错内容，再定位具体文件和原因，不要预防性扩大改动范围。
3. 不要为了通过 lint 或 typecheck 而改变原有业务逻辑。
4. 不要通过删除功能、注释核心逻辑、改成空实现等方式硬过检查。

## 代码风格与检查

修改代码后，请优先考虑项目现有规范，包括但不限于：

- ESLint
- TypeScript typecheck
- import 顺序
- React 导入风格
- 文件末尾换行、空行、行尾空格等格式规则

## React / import 规则

涉及 React 文件时，优先遵循项目现有规则，例如：

- 使用 `import * as React from 'react'`
- 按项目规则排序 import
- 不因风格调整而改动组件逻辑

## 输出要求

完成修改后，请明确说明：

1. 改了哪些文件
2. 每个文件改了什么
3. 是否可能影响页面显示或功能
4. 建议执行哪些验证命令
