###### 今天 git clone 项目部老学长的工具时，出现了点问题，又新学了一点。

###### ① 报错：xx 不是内部或外部命令

可能没有全局下载该包。npm install 解决好了。

###### ② 运行 npm install 报错

![image-20260515193540446](C:\Users\Administrator\AppData\Roaming\Typora\typora-user-images\image-20260515193540446.png)

```txt
ERESOLVE unable to resolve dependency tree
Found: xx@2.10 当前版本
peer: xx@3.10 实际需要的版本
```

这就是依赖版本冲突了，在 package.json 里修改，npm install 就行

原理：npm 会根据新的 package.json 更新 package-lock.json  
然后把正确的装进 node_modules。

###### package.json，package-lock.json，node_modules三者关系：

| 文件              | 类比       | 实例                                                |
| ----------------- | ---------- | --------------------------------------------------- |
| package.json      | 购物清单   | 我要买react 18.0.0系列！ (版本18.0.1 ~ 18.0.x 都行) |
| package-lock.json | 购物小票   | 实际买到了 18.3.1                                   |
| node_modules      | 家里的仓库 | 东西真正放进来了                                    |

###### package-lock.json：

这里的package-lock.json 就是用来锁版本，取一致的！一般都要 git 上传。  
这样以后别人协作部署时，我这个版本跑不起来，就可以去看你的 package-lock.json 你实际用的是哪个版本了，解决了“我电脑能跑，你电脑不能跑”的问题。

###### 总结：

| 文件              | 作用                       |
| ----------------- | -------------------------- |
| package.json      | 项目依赖需求，别乱删！！！ |
| package-lock.json | 实际安装锁定记录           |
| node_modules      | 实际下载下来的依赖文件夹   |

以后改依赖版本时，直接 npm install，如果还是有冲突ERESOLVE，可把后两个删了再重新 install。

Git Bash / macOS / Linux 终端写法：

```bash
rm -f package-lock.json
rm -rf node_modules
npm install
```

powershell写法：

```powershell
Remove-Item node_modules 
-Recurse -Force -ErrorAction SilentlyContinue
npm install
```
