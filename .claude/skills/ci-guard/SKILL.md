# CI Guard

当任务与以下内容相关时使用本技能：

- GitHub CI 报错
- ESLint 报错
- TypeScript typecheck 报错
- build 失败
- import 顺序、React 导入风格、格式规则修复

## 执行原则

1. 只修复当前报错涉及的问题。
2. 只修改报错涉及的文件，不扩散到无关文件。
3. 以最小改动为原则，不顺手重构，不大范围格式化。
4. 不改变原有业务逻辑，不通过删除功能或注释核心逻辑来硬过检查。

## 修复流程

1. 先阅读报错内容。
2. 按报错逐个定位到具体文件和具体规则。
3. 优先做机械性修复，例如：
   - import 顺序
   - React 导入方式
   - 空行、行尾空格、文件末尾换行
   - brace-style
4. 如果涉及可能影响逻辑的修改，例如异步结构调整、type 修复、构建配置修复，保持原意不变，只做必要改动。
5. 修复完成后，明确汇总改动。

## 项目特别注意

### React / import
涉及 React 文件时，优先遵循项目现有规则：

- 使用 `import * as React from 'react'`
- 按项目规则排序 import
- 不改组件 JSX 逻辑，除非任务明确要求

### 修 lint / typecheck 时禁止的行为
不要为了通过检查而做以下事情：

- 删除原有功能
- 注释掉核心逻辑
- 把原实现改成空实现
- 为了消除报错而改变页面行为或数据流
- 扩大修改范围去“顺手统一风格”

## 输出格式要求

完成后请输出：

1. 修改了哪些文件
2. 每个文件修复了什么问题
3. 是否有可能影响页面显示或功能
4. 建议执行的验证命令，例如：

pnpm lint
pnpm typecheck
pnpm build
